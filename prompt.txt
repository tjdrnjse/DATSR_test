# Role
당신은 PyTorch 프레임워크와 모델 배포 최적화에 정통한 시니어 AI 엔지니어입니다. 레거시 모델 가중치를 최신 아키텍처로 자동 변환하여 로드하는 'Robust Loader' 구현에 강점이 있습니다.

# Context & Goal
Python 3.8, PyTorch 1.7.1 기반의 DATSR 모델 코드를 **Python 3.10, PyTorch 2.1.0**으로 마이그레이션하고, **DCNv2를 `torchvision.ops`로 교체함에 따라 발생하는 가중치 이름 불일치 문제를 자동으로 해결하는 로더**를 구축해야 합니다.

# Algorithm Specification (Automatic Weight Loading)
1. **DCNv2 Key Mapping:** - 기존 DCNv2(C++)는 `weight`, `bias` 외에도 `conv_offset` 등의 내부 이름을 가집니다. 
   - `torchvision.ops.deform_conv2d`를 래핑한 신규 모델의 경우, 내부 레이어 구조(예: `.conv.weight`)가 달라질 수 있습니다.
   - 로더는 `state_dict`를 분석하여 DCN 관련 키 패턴을 감지하고, 신규 모델 레이어 이름으로 자동 변환(String replacement/Regex)해야 합니다.
2. **Strictness Management:** - `strict=False`로 로드하되, 로드되지 않은 키(Missing)와 가중치 파일엔 있지만 모델엔 없는 키(Unexpected)를 로그로 상세히 출력하여 개발자가 확인할 수 있게 합니다.

# Development Process & Instructions

## Phase 1: `torchvision` 기반 DCNv2 Wrapper 구현
- `torchvision.ops.deform_conv2d`를 사용하여 기존 DCNv2와 입출력이 동일한 `ModulatedDeformConv2d` 클래스를 작성하세요.
- 이때, 내부 `nn.Conv2d`나 파라미터 변수명을 원본 가중치 파일의 키 이름과 최대한 유사하게 명명하여 매핑 난이도를 낮추세요.

## Phase 2: Robust Weight Loader 함수 구현 (핵심 추가 사항)
- `load_robust_state_dict(model, checkpoint_path)` 함수를 작성하세요.
- 이 함수는 다음 기능을 포함해야 합니다:
    1. `torch.load(..., weights_only=False)`를 사용하여 체크포인트를 읽습니다.
    2. 체크포인트 내 `params` 또는 `state_dict` 계층을 자동 탐색합니다.
    3. **자동 키 변환 로직:** 구형 DCNv2의 특정 이름 패턴(예: `.dcn.`)을 감지하면 신규 Wrapper 구조에 맞는 이름으로 Key 이름을 강제 변경합니다.
    4. 변환된 `state_dict`를 모델에 `strict=False`로 주입하고, 결과 리포트를 반환합니다.

## Phase 3: DATSR 아키텍처 업데이트 및 통합
- Phase 1의 Wrapper를 RDA 모듈에 이식하고, Phase 2의 로더를 통해 모델이 성공적으로 초기화되는지 확인합니다.

## Phase 4: Low-Cost Dummy Validation
- CPU 환경에서 Dummy 텐서를 생성하고, 가상의 구버전 `state_dict` (딕셔너리)를 만들어 `load_robust_state_dict` 함수가 이름을 성공적으로 바꾸어 로드하는지 테스트하는 `test_robust_load.py`를 작성하세요.

## Phase 5: Cleanup Script
- 검증 완료 후 테스트 파일을 삭제하는 `cleanup.py`를 제공하세요.
